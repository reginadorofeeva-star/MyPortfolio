# 🚀 Развертывание на Vercel

## Быстрый старт (3 минуты)

### Метод 1: Через GitHub (РЕКОМЕНДУЕТСЯ)

#### Шаг 1: Загрузите проект на GitHub
```bash
# Инициализируйте git (если еще не сделали)
git init
git add .
git commit -m "Initial commit: Regina Portfolio"

# Создайте репозиторий на GitHub и подключите его
git remote add origin https://github.com/ваш-username/regina-portfolio.git
git branch -M main
git push -u origin main
```

#### Шаг 2: Подключите к Vercel
1. Перейдите на [vercel.com](https://vercel.com)
2. Войдите через GitHub
3. Нажмите **"Add New Project"**
4. Выберите ваш репозиторий `regina-portfolio`
5. Vercel автоматически определит настройки (Vite)
6. Нажмите **"Deploy"**
7. ✅ Готово! Ваш сайт доступен по адресу `your-project.vercel.app`

**Автоматические обновления:**
- Каждый `git push` автоматически обновит сайт
- Production деплоится из ветки `main`
- Preview деплои создаются для всех веток

### Метод 3: Drag & Drop (самый простой)

1. Соберите проект:

2. Перейдите на [vercel.com/new](https://vercel.com/new)

## Настройка кастомного домена

### Шаг 1: В панели Vercel
1. Откройте ваш проект на Vercel
2. Перейдите в **Settings → Domains**
3. Нажмите **"Add Domain"**
4. Введите ваш домен (например: `regina-portfolio.ru`)

### Шаг 2: Настройте DNS
Vercel покажет DNS записи, которые нужно добавить:

**Для корневого домена (regina-portfolio.ru):**
```
Type: A
Name: @
Value: 76.76.21.21
```

**Для www (www.regina-portfolio.ru):**
```
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

### Шаг 3: Подождите
DNS изменения могут занять до 48 часов (обычно 1-2 часа).

## Доступ из России 🇷🇺

### ⚠️ Важно про Vercel и РФ

Vercel **иногда** работает без VPN, но не всегда стабильно:
- ✅ Сам сайт обычно доступен
- ⚠️ Административная панель может требовать VPN
- ⚠️ Скорость может быть медленнее чем на российском хостинге

### Решение для стабильного доступа из РФ:

#### Вариант A: Cloudflare + Vercel
1. Добавьте домен в Cloudflare (бесплатно)
2. Настройте DNS через Cloudflare
3. Включите Cloudflare Proxy (оранжевое облачко)
4. ✅ Cloudflare будет кэшировать контент в РФ

#### Вариант B: Используйте .ru домен
- Домены в зоне .ru обычно быстрее работают в России
- Купите на REG.RU или Timeweb
- Привяжите к Vercel

#### Вариант C: Комбинированный подход
- Используйте Vercel для разработки
- Для production используйте Timeweb + GitHub Actions для деплоя
